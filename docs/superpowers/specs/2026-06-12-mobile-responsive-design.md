# 移动端响应式优化设计

## 概述

参考 grtsinry43 博客的移动端设计，对本博客进行四方面的移动端优化。所有改动仅影响 `<md` 断点（768px 以下），桌面端体验保持不变。

## 1. 导航栏 — 全屏覆盖式汉堡菜单

### 现状
- 导航链接用 `hidden md:flex`，移动端完全无导航入口

### 改动

**移动端 Header（<768px）：**
- 左侧：头像（40x40 圆形）作为菜单触发器 + 旁边文字 "Hazenix的后端札记"
- 右侧：搜索图标 + 主题切换
- 点击头像打开全屏覆盖式导航面板

**全屏覆盖面板：**
- 全屏展开，白色/深色背景
- 右上角 ✕ 关闭按钮
- 导航项大字居中排列：首页、文章（展开显示分类）、标签、树洞、友链、更多
- 底部放：登录/用户头像入口
- 滑入/淡入过渡动画

## 2. 首页 Hero — 垂直堆叠

### 现状
- `text-5xl`（~3rem）在小屏过大
- Hero 标题和 UserCard 水平排列

### 改动（<768px）
- 布局改为垂直居中：头像 → 标题 → 英文副标题 → 中文副标题
- 标题字号 `text-2xl`（~1.5rem）
- 副标题 `text-base`（~1rem）
- UserCard 信息整合进 Hero 区域

## 3. 文章详情页 — TOC 前置 + 内容后置

### 现状
- 侧边栏（TOC、标签、相关文章）通过 `isMobile` 隐藏
- 左侧固定互动按钮隐藏

### 改动（<768px）

**目录：** 在文章正文前插入可折叠 TOC
- 默认收起，显示 "📑 目录 ▼"
- 点击展开完整目录树
- 点击目录项跳转到对应标题

**互动按钮：** 移至文章正文结尾处
- 水平排列：❤️ 点赞 | ⭐ 收藏 | ↗️ 分享
- 带浅色背景区分，自然融入阅读流程

**标签 & 相关文章：** 移至文章底部（评论区上方）
- 标签以圆角 chip 横向排列
- 相关文章以链接列表展示

## 4. 页脚 — 卡片式布局

### 现状
- 3 列 grid 折叠为 1 列，间距紧凑（4px）
- 备案信息居中，小屏可能溢出

### 改动（<768px）
- 三组链接改为独立圆角卡片（`border-radius: 10px`），浅灰背景
- 每组内链接横向排列（flex row + gap），节省垂直空间
- 备案和版权改为左对齐
- padding 和段间距增大（6→12，4→12）

## 不变的部分

- 文章卡片布局（用户确认当前设计 OK）
- 桌面端所有布局和组件
- 色彩体系和暗色模式
- 代码高亮主题
