# 后端安全漏洞审计报告

## 文档信息

- **审计日期**: 2024年
- **项目名称**: HazenixBlogProject
- **审计范围**: 后端代码（blog-hazenix-top模块）

## 执行摘要

本次安全审计发现了**15个安全漏洞**，其中：
- **高危漏洞**: 5个
- **中危漏洞**: 7个
- **低危漏洞**: 3个

主要安全风险集中在：

3. 文件上传缺少安全验证
4. 依赖库存在已知漏洞
5. 输入验证不完善

## 1. 高危漏洞

### 1.1 密码加密算法不安全（高危）

**位置**: 
- `UserServiceImpl.java:97, 139, 608, 618`
- `GenerateMD5PasswordTest.java:10`

**问题描述**:
使用MD5算法对密码进行哈希处理。MD5已被证明不安全，容易被暴力破解和彩虹表攻击。

**代码示例**:
```java
String processedPassword = DigestUtils.md5DigestAsHex(userLoginDTO.getPassword().getBytes());
```

**风险等级**: 🔴 **高危**

**影响**:
- 用户密码容易被破解
- 数据库泄露时，攻击者可以快速破解密码
- 无法抵御现代密码破解技术

**修复建议**:
1. 使用BCrypt、Argon2或PBKDF2等安全的密码哈希算法
2. 推荐使用Spring Security的`BCryptPasswordEncoder`
3. 设置合适的加密强度（cost factor ≥ 12）

**修复示例**:
```java
// 使用`BCryptPasswordEncoder`编码器进行编码
// 创建编码器
// 编码器调用`String encode(String password)`方法

@Autowired
private BCryptPasswordEncoder passwordEncoder;

// 注册时
String encodedPassword = passwordEncoder.encode(userLoginDTO.getPassword());

// 登录时
if (!passwordEncoder.matches(userLoginDTO.getPassword(), user.getPassword())) {
    throw new BussinessException(ErrorCode.A01002, MessageConstant.EMAIL_OR_PASSWORD_ERROR);
}
```

### 1.2 密码长度验证缺失（高危）

**位置**: `UserServiceImpl.java:612-614`

**问题描述**:
密码长度验证代码被注释掉，允许用户设置任意长度的密码（包括空密码或极短密码）。

**代码示例**:
```java
//        if (userDTO.getNewPassword().length() < 3 || userDTO.getNewPassword().length() > 20) {
//            throw new BaseException(ErrorCode.A010XX, "密码长度必须在3-20个字符之间");
//        }
```

**风险等级**: 🔴 **高危**

**影响**:
- 允许弱密码，增加账户被破解的风险
- 不符合密码安全策略

**修复建议**:
1. 启用密码长度验证（建议8-128个字符）
2. 添加密码复杂度要求：
   - 至少包含大小写字母、数字和特殊字符中的三种
   - 不能与用户名、邮箱相同
   - 不能是常见弱密码

**修复示例**:
```java
if (userDTO.getNewPassword() == null || userDTO.getNewPassword().length() < 8 || userDTO.getNewPassword().length() > 128) {
    throw new BussinessException(ErrorCode.A01012, "密码长度必须在8-128个字符之间");

// 添加密码复杂度验证
if (!isPasswordStrong(userDTO.getNewPassword())) {
    throw new BussinessException(ErrorCode.A01013, "密码必须包含大小写字母、数字和特殊字符");
```

### 1.3 CORS配置过于宽松（高危）

**位置**: `CorsConfig.java:15-17`

**问题描述**:
CORS配置允许所有来源（`*`）、所有请求头和所有请求方法，存在严重的安全风险。

**代码示例**:
```java
corsConfiguration.addAllowedOrigin("*");//1. 设置访问源地址
corsConfiguration.addAllowedHeader("*");//2. 设置访问源请求头
corsConfiguration.addAllowedMethod("*");//3. 设置访问源请求方法
```

**风险等级**: 🔴 **高危**

**影响**:
- 任何网站都可以通过JavaScript访问API
- 容易受到CSRF攻击
- 敏感数据可能被恶意网站窃取

**修复建议**:
1. 明确指定允许的源地址（白名单）
2. 限制允许的请求头
3. 限制允许的请求方法
4. 设置`allowCredentials`为true时，不能使用`*`作为源

**修复示例**:
```java
@Value("${cors.allowed-origins}")
private List<String> allowedOrigins;

@Bean
public CorsFilter corsFilter(){
    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    CorsConfiguration corsConfiguration = new CorsConfiguration();
    
    // 只允许配置的源
    corsConfiguration.setAllowedOrigins(allowedOrigins);
    
    // 只允许必要的请求头
    corsConfiguration.setAllowedHeaders(Arrays.asList(
        "Content-Type", 
        "Authorization", 
        "X-Requested-With"
    ));
    
    // 只允许必要的请求方法
    corsConfiguration.setAllowedMethods(Arrays.asList(
        "GET", "POST", "PUT", "DELETE", "OPTIONS"
    ));
    
    // 允许携带凭证
    corsConfiguration.setAllowCredentials(true);
    
    // 预检请求缓存时间
    corsConfiguration.setMaxAge(3600L);
    
    source.registerCorsConfiguration("/**", corsConfiguration);
    return new CorsFilter(source);
}
```

### 1.4 文件上传缺少安全验证（高危）

**位置**: `WebController.java:27-39`

**问题描述**:
文件上传接口缺少以下安全验证：
- 文件类型验证
- 文件大小限制
- 文件名安全检查
- 文件内容验证（防止恶意文件）

**代码示例**:
```java
@PostMapping("/upload")
public Result upload(MultipartFile file) {
    log.info("文件上传：{}",file.getOriginalFilename());
    String url;
    try {
        url = aliOssUtil.upload(file.getBytes(),file.getOriginalFilename());
    } catch (IOException e) {
        log.error("错误信息:{}", e);
        throw new FailUploadException(ErrorCode.B00004, MessageConstant.UPLOAD_FAILED);
    }
    return Result.success(url);
}
```

**风险等级**: 🔴 **高危**

**影响**:
- 可能上传恶意文件（如PHP、JSP等可执行脚本）
- 可能上传超大文件导致DoS攻击
- 文件名可能包含路径遍历字符（`../`）
- 可能上传病毒或恶意软件

**修复建议**:
1. 验证文件类型（白名单机制）
2. 限制文件大小（如10MB）
3. 对文件名进行安全处理（移除特殊字符，生成唯一文件名）
4. 验证文件内容（MIME类型检查）
5. 对图片文件进行二次验证（检查文件头）
6. 添加访问权限控制（需要登录）

**修复示例**:
```java
@PostMapping("/upload")
public Result upload(@RequestParam("file") MultipartFile file) {
    // 1. 验证文件是否为空
    if (file == null || file.isEmpty()) {
        throw new BussinessException(ErrorCode.A00001, "文件不能为空");
    }
    
    // 2. 验证文件大小（10MB）
    if (file.getSize() > 10 * 1024 * 1024) {
        throw new BussinessException(ErrorCode.A00001, "文件大小不能超过10MB");
    }
    
    // 3. 验证文件类型（白名单）
    String originalFilename = file.getOriginalFilename();
    String extension = getFileExtension(originalFilename);
    List<String> allowedExtensions = Arrays.asList("jpg", "jpeg", "png", "gif", "webp");
    if (!allowedExtensions.contains(extension.toLowerCase())) {
        throw new BussinessException(ErrorCode.A00001, "不支持的文件类型");
    }
    
    // 4. 验证MIME类型
    String contentType = file.getContentType();
    if (!isAllowedContentType(contentType)) {
        throw new BussinessException(ErrorCode.A00001, "不支持的文件类型");
    }
    
    // 5. 生成安全的文件名
    String safeFilename = generateSafeFilename(originalFilename);
    
    try {
        // 6. 验证文件内容（检查文件头）
        byte[] fileBytes = file.getBytes();
        if (!isValidImageFile(fileBytes)) {
            throw new BussinessException(ErrorCode.A00001, "文件内容验证失败");
        }
        
        url = aliOssUtil.upload(fileBytes, safeFilename);
    } catch (IOException e) {
        log.error("文件上传失败: {}", e);
        throw new FailUploadException(ErrorCode.B00004, MessageConstant.UPLOAD_FAILED);
    }
    
    return Result.success(url);
}

private String generateSafeFilename(String originalFilename) {
    String extension = getFileExtension(originalFilename);
    // 使用UUID生成唯一文件名，防止文件名冲突和路径遍历
    return UUID.randomUUID().toString() + "." + extension;
}
```

### 1.5 Fastjson存在已知安全漏洞（高危）

**位置**: `pom.xml:23, 50-52`

**问题描述**:
项目使用了`fastjson 1.2.76`版本，该版本存在多个已知的安全漏洞（如反序列化漏洞、远程代码执行漏洞等）。

**代码示例**:
```xml
<fastjson>1.2.76</fastjson>
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>fastjson</artifactId>
    <version>${fastjson}</version>
</dependency>
```

**风险等级**: 🔴 **高危**

**影响**:
- 可能被利用进行远程代码执行（RCE）
- 反序列化漏洞可能导致服务器被完全控制
- 数据泄露风险

**修复建议**:
1. **强烈建议**：升级到最新版本（fastjson2）或使用其他JSON库（如Jackson、Gson）
2. 如果必须使用fastjson，至少升级到1.2.83+版本
3. 避免反序列化不可信的数据
4. 使用`JSON.parseObject()`时指定`ParserConfig`并禁用`AutoType`

**修复示例**:
```xml
<!-- 方案1: 升级到fastjson2 -->
<dependency>
    <groupId>com.alibaba.fastjson2</groupId>
    <artifactId>fastjson2</artifactId>
    <version>2.0.43</version>
</dependency>

<!-- 方案2: 使用Jackson（Spring Boot默认） -->
<!-- 移除fastjson依赖，使用Spring Boot自带的Jackson -->
```

### 2.2 JWT签名密钥可能不够强（中危）

**位置**: `JwtUtil.java:34, 52`

**问题描述**:
JWT签名密钥从配置文件中读取，如果密钥不够强（长度不足、复杂度不够），可能被暴力破解。

**风险等级**: 🟡 **中危**

**影响**:
- JWT可能被伪造
- 攻击者可能生成有效的token

**修复建议**:
1. 确保JWT密钥至少256位（32字节）
2. 使用随机生成的强密钥
3. 定期轮换密钥
4. 使用环境变量存储密钥，不要提交到代码仓库

### 2.3 缺少CSRF保护（中危）

**问题描述**:
项目未实现CSRF（跨站请求伪造）保护机制。

**风险等级**: 🟡 **中危**

**影响**:
- 用户可能被诱导执行非预期的操作
- 恶意网站可能利用用户身份进行操作

**修复建议**:
1. 使用Spring Security的CSRF保护
2. 对于RESTful API，可以使用CSRF Token或SameSite Cookie
3. 验证Referer头

### 2.6 缺少请求频率限制（中危）

**问题描述**:
API接口缺少请求频率限制（Rate Limiting），可能被滥用进行：
- 暴力破解攻击
- DDoS攻击
- 资源耗尽攻击

**风险等级**: 🟡 **中危**

**影响**:
- 账户可能被暴力破解
- 服务器资源可能被耗尽
- 正常用户可能无法访问

**修复建议**:
1. 实现请求频率限制（如使用Redis + 令牌桶算法）
2. 对登录接口实施更严格的限制（如5次/分钟）
3. 对IP地址进行限制
4. 使用Spring Boot的`@RateLimiter`注解或第三方库

### 2.7 文章浏览量可被恶意刷量（中危）

**位置**: `ArticleController.java:198-202`

**问题描述**:
文章浏览量接口允许客户端直接指定增加的数量，且缺少验证，可能被恶意刷量。

**代码示例**:
```java
@PostMapping("/{id}/view/{count}")
public Result updateArticleView(@PathVariable Long id,@PathVariable Integer count){
    log.info("更新文章浏览数:{}",id);
    articleService.addArticleViewByMe(id,count);
    return Result.success();
}
```

**风险等级**: 🟡 **中危**

**影响**:
- 浏览量数据不准确
- 可能影响文章排序和推荐算法

**修复建议**:
1. 移除`count`参数，每次只增加1
2. 使用IP地址或用户ID进行去重
3. 使用Redis缓存，定期批量更新数据库
4. 添加时间间隔限制（如同一IP在1分钟内只计算1次）

**修复示例**:
```java
@PostMapping("/{id}/view")
public Result updateArticleView(@PathVariable Long id, HttpServletRequest request){
    String ip = getClientIp(request);
    String key = "article:view:" + id + ":" + ip;
    
    // 使用Redis检查是否在1分钟内已访问
    if (redisTemplate.hasKey(key)) {
        return Result.success(); // 已访问，不重复计数
    }
    
    // 设置1分钟过期
    redisTemplate.opsForValue().set(key, "1", 60, TimeUnit.SECONDS);
    
    // 异步更新浏览量
    articleService.incrementArticleView(id);
    return Result.success();
}
```

## 3. 低危漏洞

### 3.1 错误信息可能泄露系统信息（低危）

**位置**: `GlobalExceptionHandler.java`及相关异常处理

**问题描述**:
错误信息可能包含系统内部信息（如堆栈跟踪、数据库错误等），虽然已有全局异常处理，但需确保不向客户端暴露敏感信息。

**风险等级**: 🟢 **低危**

**修复建议**:
1. 确保生产环境不返回详细错误信息
2. 使用错误码代替详细错误信息
3. 记录详细错误到日志，但只返回通用错误信息给客户端

---

### 3.2 缺少安全响应头（低危）

**问题描述**:
HTTP响应中缺少安全相关的响应头，如：
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Strict-Transport-Security`（如果使用HTTPS）
- `Content-Security-Policy`

**风险等级**: 🟢 **低危**

**修复建议**:
在Spring Boot配置中添加安全响应头：

```java
@Configuration
public class SecurityHeadersConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new HandlerInterceptor() {
            @Override
            public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
                response.setHeader("X-Content-Type-Options", "nosniff");
                response.setHeader("X-Frame-Options", "DENY");
                response.setHeader("X-XSS-Protection", "1; mode=block");
                response.setHeader("Strict-Transport-Security", "max-age=31536000; includeSubDomains");
                return true;
            }
        });
    }
}
```
